Cybersecurity Posture — Executive Summary

This report covers the cybersecurity health of Palmetto Clean Technology's cloud infrastructure as of 16 July 2026. The assessment spans 19 assets across Google Cloud Platform and Amazon Web Services, drawn from the palmetto-solar-d6a95 GCP project and the us-east-2 AWS environment. Assets include the Palmetto Solar customer platform (Firebase, App Engine, Firestore), mobile apps (Android, iOS, Web), data stores (BigQuery, Cloud Storage), IAM identities, and core AWS network infrastructure.

XORCISM matched 306 unique CVEs against these assets using the NVD vulnerability database (28,359 entries, current to July 2026). Of those, 51 are rated Critical (CVSS ≥ 9.0) and 124 are rated High. The predominant risk theme is the Node.js ecosystem — the bulk of critical-severity findings target the vm2 sandbox library and Node.js runtime itself, both of which underpin the App Engine backend and Firebase Web platform. A second cluster targets Google BigQuery authorization controls, relevant to Palmetto's analytics datasets which contain customer PII.

The positive headline is that zero CVEs are flagged as CISA Known Exploited (KEV), and no assets are rated SSVC Act or Attend — meaning no evidence of active exploitation in the wild has been associated with Palmetto's specific exposure profile. This does not reduce urgency on the Node.js findings, but it does mean there is no emergency requiring immediate incident response.

The AWS footprint visible in this report is thin — only default VPC plumbing was captured in the export. If workloads (EC2, RDS, Lambda) exist in us-east-2 or other regions, they are not yet represented in this risk model and should be added in the next inventory cycle.

Recommended action
Initiate a Node.js dependency audit across App Engine and Firebase Web; rotate all service account keys; apply BigQuery IAM review. These three actions address the majority of scored critical and high findings. No emergency response posture is required at this time.
